## 🔧 Two Options to Run the Platform

### Option 1: Docker Compose (Recommended - Easiest)
Docker handles all dependencies including chromadb:

```bash
# Navigate to project root
cd automind-intelligence-platform

# Set your Groq API key
$env:GROQ_API_KEY = "your-api-key-here"

# Start all services
docker-compose up --build

# Services will be available at:
# Backend:  http://localhost:8000
# Frontend: http://localhost:5173
# API Docs: http://localhost:8000/docs
```

**What happens:**
- Backend starts with all dependencies (including chromadb)
- Frontend builds and runs
- Images generated by backend are stored in `./database/generated_images/`
- Frontend fetches reports with image URLs and displays them

### Option 2: Local Development (Windows - Limitations)
If Docker is not available:

1. **Image generation works perfectly** ✅
   ```bash
   # Test only image generation
   python backend/test_image_only.py
   ```

2. **Full pipeline requires chromadb workaround**
   - Memory storage not available on Windows (chromadb build error)
   - Images still generate and can be served
   - Consider using Docker for full functionality

## 🎨 Image Generation Pipeline

```
User Prompt
    ↓
Prompt Enhancement (LLM)
    ↓
Research (Web search + LLM)
    ↓
Parallel Processing:
    ├─ Report Generation
    └─ Image Generation (Pollinations.ai API)
    ↓
Memory Storage (ChromaDB)
    ↓
Frontend receives:
    ├─ report_text
    ├─ image_filename
    └─ image_url: "/images/{filename}.png"
    ↓
React Component displays image + report
```

## 🔍 Troubleshooting

### Images not showing in frontend?

**Check 1: Are images being generated?**
```bash
ls -la backend/database/generated_images/
# Should show .png files with timestamps
```

**Check 2: Is image URL correct in response?**
```bash
# Check API response includes image_url
curl http://localhost:8000/api/reports | grep image_url
```

**Check 3: Can static file endpoint be accessed?**
```bash
# Should return 200 if image exists
curl -I http://localhost:8000/images/{filename}.png
```

**Check 4: Browser console errors?**
- Open DevTools (F12)
- Check Network tab for failed image requests
- Check Console for JavaScript errors
